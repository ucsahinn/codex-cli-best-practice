# Roadmap

This roadmap keeps the fork useful without turning the README into a backlog.

## Done In v0.1.0

- Independent EN/TR welcome page.
- Turkish onboarding guide.
- Release checklist.
- Research notes tied to current Codex docs.
- Dependency-free docs validator.
- GitHub Actions docs guard.
- Istanbul agent-to-skill demo.

## Done In v0.1.1

- Public-readiness, Windows, and agent-security docs.
- Safer shared config defaults and local hook ignores.
- Stronger docs validator for links, JSON, mojibake, local paths, stale keys, and hook logs.
- GitHub PR template and CI secret-scan job.
- Refreshed README and Turkish guide with clearer visual navigation.

## Next

- Add screenshots of the rendered GitHub landing page after the public release is verified.
- Package the weather demo as a small plugin example.
- Add a second skill that materializes a project-specific `AGENTS.md`.
- Add optional markdown formatting checks if the repo adopts a formatter.

## Later

- Add a Turkish release announcement template.
- Add a docs ownership table when contributors are known.
- Add a small compatibility matrix for Codex CLI, Codex IDE, and cloud workflows.

## Not Planned

- Runtime application code.
- Mandatory external dependencies for local validation.
- Auto-publish workflows that create tags or releases without human review.
